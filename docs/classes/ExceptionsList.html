Indigo Frost Exceptions List

Summary

Indigo Frost is a complicated tool, and as such is exception-driven in its error handling. These errors can be caught by the programmer in the case of deep involvement in the engine, or otherwise are handled by the management-tier classes.

All of these generic exceptions are contained in FrostExceptions.h, and are empty classes unless otherwise specified.

Exception NameDescription
NotImplementedException Thrown when a function is called that is not yet implemented in the engine. This mostly comes in handy in development, but may also be used when adding new types of collision geometry.
DivByZeroException Thrown in places where dividing by zero is likely to occur (like vector normalization) upon an attempt to divide by zero. This catches such errors pre-OS code, and so the engine can continue running in the case of flukes.
IndexOutOfBoundsException Contains one property - int indexGiven, which holds the index provided to a list structure that broke bounds constraints. The only constructor for this exception requires this number.
ZeroMagnitudeException Derives from DivByZeroException. Happens when a normalized vector data structure (Vect3Normal, Quaternion) is assigned with no magnitude, and thus cannot be normalized.
SingularMatrixException Thrown on attempting to invert a singular, or non-invertable, matrix. I predict the biggest cause of this exception will be invalid inertia tensors in developer-fed code to the engine. When I was designing the first round of this engine, my initial inertia tensor generating script always spit out singular matricies. Whoops.
DuplicateActionException An action which must only occur once (such as assigning an IPhysicsObject to a ICollisionGeometry instance) has been attempted two or more times.

I anticipate more entries to this list as the engine grows. Parts of the engine may specify their own exceptions as well, to be more specific. These exceptions are considered global, and are accessible to math, physics, and management tiers of the engine.
