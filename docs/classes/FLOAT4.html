Summary

128-bit wide struct holding four floats, named _x, _y, _z, and _w. This class has no methods. Located in FloatStructs.h, under the namespace Frost.

Since this struct is 128-bit aligned, it can easily integrate into HLSL buffers. I'm assuming the same goes for OpenGL - but, you caught me, I'm a Microsoft fan ;)

The definition, found in FloatStructs.h, is:

struct FLOAT4 { float _x; float _y; float _z; float _w; };

Properties:

_x First float in the structure (x/r component)
_y Second float in the structure (y/b component)
_z Third float in the structure (z/b component)
_w Fourth float in the structure (w/a component)

Methods:

The FLOAT4 struct has no methods.

Usage Notes:

I built the FLOAT3, FLOAT4 and FLOAT4X4 structures to play nicely with similar mathematical structures in other libraries. FLOAT4, for instance, plays nicely with XNAMath's XMFLOAT4 and Direct3D's D3DXVECTOR4.

Any other 128-bit float4 structure can be casted to a FLOAT4 like so (though it is kind of hairy by itself):

FLOAT4 myFloat4 = *reinterpret_cast<FLOAT4*>(&other_4_float_var);
