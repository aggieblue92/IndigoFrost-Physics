Summary

512-bit wide struct holding four FLOAT4s, in union with a 4x4 array of floats, or 16 floats - whatever floats your boat, or whatever is convinient for you. This class has no methods, like FLOAT4 and FLOAT3. Like those other two, it is located in FloatStructs.h.

This struct is 512-bit aligned, so it should fit nicely in with HLSL buffers.

The definition is:

struct FLOAT4X4 { union { float m[4][4]; struct { float _11, _12, _13, _14; float _21, _22, _23, _24; float _31, _32, _33, _34; float _41, _42, _43, _44; }; FLOAT4 rows[4]; }; };

The matrix can be retrieved as a 2D array (m), 16 individual floats (_11, etc.) or as an array of the columns (rows). The union makes this possible.

Properties:

m 2D array, containing 4 rows and 4 columns, representing the struct as a matrix.
_11, ..., _[m][n] A 32-bit float representing the [m]th row and [n]th column of the matrix.
rows An array of 4 FLOAT4 objects, representing the struct as an array of row vectors.

Methods:

This struct has no methods.

Usage Notes:

Like FLOAT3, FLOAT4, and the derived classes of such, I built this structure so that it and all immediately derived mathematical classes would play nicely with other libraries. I haven't tested this one as extensively as I have the other two, but so far conversions to and from the XNAMath library MATRIX class have been smooth, and likewise for loading into HLSL buffers using Direct3D 11.
