Summary

Represents a 4x4 matrix, used to represent a linear transformation in 3D space, holding rotational and translational data. The upper-left 3x3 part of the matrix holds rotational information, the three upper values on the right column hold translational data, and the bottom row makes it a square matrix that can interact with Vect3 and Matrix instances.

Matrix(const Quaternion& quat)

Creates a rotation matrix that represents the same rotation that the given Quaternion object represents.

Matrix(const Vect3& translation)

Creates a transformation matrix that performs a translation. The translation to perform is given by the provided Vect3 object

Matrix(const Quaternion& rotation, const FLOAT3& translation)

Creates a transformation matrix comprised of the given orientation and translation in 3D space. Fun fact - rotation matrices only affect the upper-left 3X3 part of the matrix, while translations only affect the top 3 values in the fourth column. So this is a glorified method that just combines the results of Matrix(const Quaternion&) and Matrix(const Vect3&).

rotation A quaternion representing an orientation in 3D space.
translation A 3D vector representing a translation in 3D space

Matrix GetInverse() const

Returns the inverse of the matrix object.
This is useful because if a matrix represents a transformation in 3D space, then the inverse of that matrix represents the opposite transformation - i.e., how to un-do that transformation.

Matrix GetTranspose() const

Returns the transpose of the matrix object.
At the time of writing this physics engine, one of the optimizations I did not do is take advantage of the fact that for a rotational matrix with no translation information, the inverse of that matrix is simply its transpose. Linear algebra voodo.

void getOrientationAndPosition(Vect3& o_pos, Quaternion& o_orientation) const

Decomposes the matrix into a rotation and a translation, and then assigns those values to the passed parameters. Think of it as the inverse function to the Matrix(const Quaternion&, const FLOAT3&) constructor.

o_pos A Vect3 object that will be assigned the translation associated with this matrix
o_orientation A Quaternion object that will be assigned the rotation associated with this matrix

Quaternion getOrientation() const

Returns the orientation that is associated with this matrix. A matrix formed from the returned quaternion would be (floating point error aside) identical to this matrix, assuming this matrix held no translational data.

Vect3 getPosition() const

Returns the position that is associated with this matrix. A matrix formed from the returned vector would be (floating point error aside) identical to this matrix, assuming this matrix held no rotational data.

FLOAT3 TransformCoord(const FLOAT3& coord) const

Transforms a coordinate vector - includes all rotation and translation operations when dealing with the vector. Use this for dealing with points you wish to transform between spaces.

coord The point to transform by this Matrix object.

FLOAT3 TransformDirn(const FLOAT3& dirn) const

Transforms a direction vector - includes only rotational operations when dealing with the vector. Use this for dealing with directions you wish to transform between spaces.

dirn The direction to transform by this Matrix object.

Vect3Normal TransformNormal(const FLOAT3& dirn) const

Does the exact same thing as the above function, but since it returns a Vect3Normal object, the returned direction will be a unit vector. In reality, this function is just taking up a few extra lines of code, but who knows? Somebody might use it!

Note to reader - I know that's a terrible reason to include functionality. That's "feature creep" even at this low level of the engine. I would not be surprised if this function is retired.

dirn The direction to transform by this Matrix object.

Usage Notes:

When using DirectX, take care - Indigo Frost was written with right (column) vector multiplication in mind, but DirectX uses left (row) vector multiplication. You may have to transpose vectors before using them in certain situations with DirectX objects.
