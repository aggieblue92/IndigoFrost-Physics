Summary

Vect3Normal is simply an extension of Vect3, with the only difference being that it is guaranteed to be a normal vector (have a magnitude of 1.0). Any time a function is called that can modify the Vect3Normal instance, the normalization process is performed. So, any time you use it in your code, you are guaranteed a normal vector (except in one instance, outlined in the notes below). This is especially useful in vector mathematics.

Vect3Normal is held in Vect3Normal.h

Usage Notes:

You'll notice in Vect3Normal.h, there are some operators defined in the private: section of the class. They are all marked by comments specifying FORBIDDEN! This is because these operations (namely +=, /=, */, and -=) involve modifying the existing Vect3 object, which will almost always result in an invalid Vect3Normal object, and almost never makes sense to do anyways.

All assignment operations perform normalization as part of the process, and all arithmatic operations are derived from Vect3. As such, so long as you are not directly modifying the member data of the class, there is no possible way to disturb the normal status of the Vect3Normal object.

Again: it is possible to corrupt the data by directly modifying _x, _y, or _z.

Notice: The Frost::MathConstants::VECTOR_UNIT_* objects are Vect3 instances, even though they are all normal vectors. This is for ease of assignment into Vect3 objects, which by far are more commonly used in outward-facing code.
